# 文件分类整理程序 - 使用指南

## 程序简介

这是一个 Python 文件分类整理程序，可以根据文件的创建时间和文件类型，自动将文件整理到结构化的文件夹中。

## 功能特点

### 1. 操作模式

#### 整理模式

对文件进行分类整理，支持两种整理方式：

- 按照时间整理（年/月/文件类型 三级目录）
- 按照类型整理（直接按文件类型分类）

#### 回滚模式（新增）

基于 log 文件撤销整理操作，可以将文件恢复到整理前的状态：

- 查看目标目录的所有整理记录
- 选择任意一次整理记录进行回滚
- 支持回滚移动操作（将文件移回源目录）
- 支持回滚复制操作（删除复制的文件）
- 可选择是否删除回滚的整理记录

#### 查看日志模式（新增）

查看整理记录和文件移动情况：

- 支持查看当前目录或指定目录的日志
- 显示每次整理的详细信息（时间、源目录、整理模式等）
- 列出每个文件/文件夹的详细移动情况（源路径、目标路径、操作类型）
- 查看完日志后可直接进入回滚模式

### 2. 整理模式

#### 按照时间整理（模式 1）

程序会在目标目录中创建三级分类文件夹：

- **第一级**：按年份分类（如：2026）
- **第二级**：按月份分类（如：03）
- **第三级**：按文件类型分类（视频、图片、文档）

#### 按照类型整理（模式 2）

程序会在目标目录中直接按文件类型创建一级文件夹：

- **图片**：所有图片格式文件
- **视频**：所有视频格式文件
- **文档**：所有文档格式文件
- **音频**：所有音频格式文件（新增）
- **应用程序**：所有应用程序安装包
- **压缩包**：所有压缩文件格式
- **其他**：未分类的其他文件

### 3. 支持的文件类型

#### 视频文件

`.mp4`, `.avi`, `.mkv`, `.mov`, `.wmv`, `.flv`, `.webm`, `.m4v`, `.mpeg`, `.mpg`, `.3gp`

#### 图片文件

`.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.tiff`, `.tif`, `.webp`, `.heic`, `.raw`, `.cr2`, `.nef`

#### 文档文件

`.pdf`, `.doc`, `.docx`, `.xls`, `.xlsx`, `.ppt`, `.pptx`, `.txt`, `.md`, `.rtf`, `.odt`, `.ods`, `.odp`

#### 音频文件（新增）

`.mp3`, `.wav`, `.flac`, `.aac`, `.ogg`, `.wma`, `.m4a`, `.ape`, `.opus`, `.aiff`

#### 应用程序文件

`.exe`, `.app`, `.msi`, `.pkg`, `.dmg`, `.deb`, `.rpm`, `.msix`（新增）

#### 压缩包文件

`.zip`, `.rar`, `.7z`, `.tar`, `.gz`, `.bz2`, `.xz`, `.iso`

### 4. 排除的文件类型

以下类型的文件不会被处理：

- 系统链接文件：`.lnk`
- 系统动态库：`.dll`, `.sys`
- 脚本文件：`.bat`, `.cmd`, `.sh`
- 其他系统文件：`.pif`, `.scr`
- 程序自身文件：动态检测程序文件名，无论程序被重命名或打包为何种格式都能正确识别并排除（v3.2 Beta 新增）
- 文档文件：`README.md`（硬编码排除，不区分大小写）
- 模块文件：`__main__.py`（不区分大小写）

### 5. 操作模式

#### 复制模式（推荐）

- 将文件**复制**到目标目录
- **保留**原始文件
- 安全，不会丢失数据

#### 移动模式

- 将文件**移动**到目标目录
- **删除**原始文件
- ⚠️ 使用前请务必备份重要文件

### 6. 文件夹处理

- 可选择是否处理源目录中的文件夹
- **按照时间整理模式**：
  - 按文件夹的创建日期分类到年/月/文件夹 目录
  - 支持递归处理所有子文件夹中的文件（新增）
  - 移动模式下，空文件夹自动移动到"空文件夹"目录（新增）
  - 选择处理文件夹时会先预览文件夹的整理路径
- **按照类型整理模式**：递归处理文件夹内所有文件，直接按类型分类

### 7. Log 文件（原整理说明文档）

- 每次整理结束后，自动在目标目录生成 `log.json` 文件
- 记录本次整理的详细信息：
  - 整理日期时间
  - 整理模式选项
  - 操作模式
  - 处理的项目列表（包含源路径和目标路径）
- **历史记录累加**：多次使用同一目标目录时，新 log 会自动包含以往所有整理记录
- **回滚功能基础**：log 文件存储了完整的整理记录，支持回滚操作

## 注意事项

### ⚠️ 重要提示

1. **移动模式风险**：使用移动模式前，请务必备份重要文件
2. **目标目录保护**：程序会自动跳过目标目录，避免递归处理
3. **文件重名处理**：如果目标文件已存在，会自动添加数字后缀（如：file\_1.pdf）
4. **程序自我保护**：程序会动态检测自身文件名并自动跳过，无论程序被重命名或打包为何种格式都能正确识别（v3.2 Beta 优化）
5. **Log 文件累加**：每次整理都会累加历史记录，不会覆盖之前的记录
6. **回滚操作**：回滚移动操作会将文件移回源目录，请确保源目录仍然存在
7. **Log 文件重要性**：log 文件是回滚功能的基础，请勿随意删除
8. **输入验证**：所有用户输入都会进行严格验证，必须输入有效值才能继续（v3.3 Beta 新增）
9. **随时退出**：在任何输入点输入 `q` 或 `quit` 可随时退出程序（v3.3 Beta 新增）

### 建议

1. **首次使用**：建议先使用复制模式测试
2. **目录选择**：建议使用"在源目录下创建新文件夹"选项
3. **备份**：处理大量文件前，建议先备份
4. **测试**：可先用少量文件测试效果
5. **查看 Log**：整理完成后可查看 `log.json` 了解详细整理情况
6. **保留 Log**：建议保留 log 文件以便需要时进行回滚操作

## 常见问题

### Q: 程序会删除我的原始文件吗？

A: 只有在"移动模式"下才会删除原始文件。默认推荐使用"复制模式"，会保留原始文件。

### Q: 如果文件已经存在怎么办？

A: 程序会自动检测重名文件，并添加数字后缀（如：file\_1.pdf, file\_2.pdf）。

### Q: 两种整理模式有什么区别？

A:

- **按照时间整理**：创建年/月/类型三级目录，适合按时间线整理文件
- **按照类型整理**：直接按类型创建一级目录，结构更简洁，适合快速分类

### Q: 支持子目录递归处理吗？

A:

- **按照时间整理模式**：
  - 默认只处理源目录的第一层内容（不递归处理子目录）
  - 可以选择是否处理第一层的文件夹
  - **v3.1 Beta 新增**：可选择递归处理所有子文件夹中的文件，移动模式下空文件夹会自动移动到"空文件夹"目录
- **按照类型整理模式**：选择处理文件夹时，会递归处理所有子文件夹中的文件

### Q: 如何停止程序？

A: 在确认步骤输入非 `y` 的字符即可取消操作。

### Q: 程序会整理它自己吗？

A: 不会。程序会动态检测自身文件名并自动跳过，无论程序被重命名或打包为何种格式（如 Windows 的 .exe 或 macOS 的 .app）都能正确识别。同时也会跳过 `README.md` 和 `__main__.py` 文件（不区分大小写）。

### Q: Log 文件可以删除吗？

A: 可以删除，但建议保留以便追踪整理历史和进行回滚操作。删除后：

- 无法查看历史整理记录
- 无法使用回滚功能
- 下次整理会重新创建新的记录（不包含之前的历史）

### Q: 可以多次整理到同一个目标目录吗？

A: 可以。程序会自动累加历史记录到 `log.json`，每次整理都会保留之前的记录。

### Q: 回滚操作是如何工作的？

A: 回滚模式会读取 log 文件中的整理记录：

- **移动操作回滚**：将文件从目标目录移回源目录
- **复制操作回滚**：直接删除目标目录中的复制文件
- 回滚完成后，可选择是否删除该条整理记录

### Q: 回滚时文件不存在怎么办？

A: 如果目标文件已被删除或移动，程序会显示警告并跳过该文件，继续回滚其他文件。

### Q: 可以回滚任意一次整理记录吗？

A: 是的。程序会列出所有历史整理记录，您可以选择任意一次进行回滚。回滚顺序是从后往前，但不影响选择任意记录。

### Q: 如果输入无效的选项会怎样？（v3.3 Beta 新增）

A: 程序会对所有用户输入进行严格验证：

- **选项输入**：必须输入有效的选项（如 1 或 2），否则会提示重新输入
- **路径输入**：必须输入有效的目录路径，不存在的目录会被拒绝
- **确认输入**：必须输入 y（确认）或 n（取消），其他输入会被拒绝
- **随时退出**：在任何输入点输入 `q` 或 `quit` 可退出程序

### Q: 如何中途退出程序？（v3.3 Beta 新增）

A: 在任何需要用户输入的步骤，输入 `q`、`quit` 或 `退出` 即可退出程序。程序会显示"程序已退出"消息并安全结束。
